# RACE Project Page Layout Notes

这个仓库仍然是单页静态站点，但页面布局已经从原始学术模板改成了更明确的论文项目页结构。下面只记录 `HTML` 和 `CSS` 层面的布局调整，不涉及正文内容本身。

## HTML 布局修改

- 页面仍然只有一个入口文件 `index.html`，没有引入新的页面、模板系统或组件框架。
- 页面顶层结构被整理为三段：
  - 固定悬浮的 `More Works` 入口
  - 主体内容 `main`
  - 页脚 `footer`
- Hero 区改成了更紧凑的首屏结构，依次放置：
  - 会标区
  - 标题区
  - 作者与机构区
  - 主操作按钮区
- 页面主体不再使用模板默认的 overview/gallery 式堆叠，而是拆成顺序 section。
- 每个主要图示都统一收拢到 `article.project-figure` 结构中，内部固定为两部分：
  - `.project-figure-media`
  - `.project-figure-caption`
- 图片区现在分成两种版式：
  - 常规整行卡片
  - `project-figure--split` 图文分栏卡片
- 分栏卡片支持 `project-figure--reverse`，用于在桌面端切换左右顺序，但不改变整体组件结构。
- 页面底部新增了一个全局图片预览容器，作为所有内容图共用的弹层壳体，避免每张图重复插入独立弹窗结构。
- `BibTeX` 区保留为独立 section，并维持标题与操作按钮的并排头部结构。

## CSS 布局修改

- 样式仍然集中在 `static/css/index.css`，没有拆出新的样式文件。
- 页面整体视觉继续沿用原模板的卡片、圆角、阴影和 Bulma 基础栅格，但对排版层次做了重排。
- Hero 区补充了会标与标题的专用布局样式，使首屏从模板默认的大块留白改成更紧凑的纵向编排。
- `.project-figure` 继续作为统一图卡容器，负责边框、圆角、阴影和基础留白。
- `.figure-narrow`、`.figure-medium`、`.figure-wide` 三档宽度控制仍然保留，用于整行图的视觉节奏区分。
- 新增 `project-figure--split` 分栏样式后，图卡在桌面端改为双列布局：
  - 图片区和图注区并排
  - 图注区垂直居中
  - 反向分栏通过顺序切换实现
- 分栏图卡中的图片区域增加了独立背景、内边距和更强的居中约束，使非超宽图不会再被机械地按整行铺开。
- 图注区在分栏模式下统一左对齐；移动端回退为单列时，再恢复为更适合窄屏阅读的居中排布。
- 图片预览弹层补充了完整的遮罩、弹窗、关闭按钮和大图容器样式，但仍复用当前页面的配色、圆角和阴影体系。
- `More Works`、回到顶部按钮、BibTeX 头部等已有浮层/操作区样式继续保留，没有改成新的组件体系。

## 响应式规则

- 在移动端断点下，分栏图卡统一回退为单列堆叠，避免横向压缩。
- 原本针对 `figure-narrow / medium / wide` 的宽度限制在小屏幕下继续回退为 `max-width: 100%`。
- 弹层在小屏幕下同步收紧边距、圆角和图片最大高度，避免遮挡过多视口。
- Hero、按钮区、下拉层和图卡内边距都保留了现有的移动端适配规则，没有引入新的断点体系。
